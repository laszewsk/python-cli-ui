Python CLI UI¶

Tools for nice user interfaces in the terminal.

Installation¶

python-cli-ui is available on Pypi and is compatible with Python 3.4 and higher.

It depends on colorama and unidecode for Windows support, and on tabulate for the info_table() function.

Note

The name of the Pypi package is python-cli-ui, but after you install it, you should use import ui to use it.

API¶

Configuration¶

ui.setup(*, verbose=False, quiet=False, color='auto', title='auto', timestamp=False)¶

Configure behavior of message functions.

Parameters:
  • verbose – Whether debug() messages should get printed
  • quiet – Hide every message except warning(), error(), and fatal()
  • color – Choices: ‘auto’, ‘always’, or ‘never’. Whether to color output. By default (‘auto’), only use color when output is a terminal.
  • title – Ditto for setting terminal title
  • timestamp – Whether to prefix every message with a time stamp
 
>>> ui.debug("this will not be printed")
<nothing>
>>> ui.setup(verbose=True)
>>> ui.debug("this will be printed")
this will be printed

Constants¶

You can use any of these constants as a token in the following functions:

  • Effects:
    • bold
    • faint
    • standout
    • underline
    • blink
    • overline
    • reset
  • Colors:
    • black
    • blue
    • brown
    • darkblue
    • darkgray
    • darkgreen
    • darkred
    • darkyellow
    • fuchsia
    • fuscia
    • green
    • lightgray
    • purple
    • red
    • teal
    • turquoise
    • white
    • yellow

Note

Each of these constants is an instance of Color.

  • Sequence of Unicode characters:

    • check: ✓ (green, replaced by ‘ok’ on Windows)
    • cross: ❌ (red, replaced by ‘ko’ on Windows)
    • ellipsis: … (no color, replaced by ‘…’ on Windows)

    You can create your own colored sequences using UnicodeSequence:

class ui.UnicodeSequence(color, as_unicode, as_ascii)¶

Represent a sequence containing a color followed by a Unicode symbol

>>> up_arrow = ui.UnicodeSequence(ui.blue, "↑", "+")
>>> ui.info(up_arrow, "2 commits")
↑ 2 commits # on Linux
+ 2 commits # on Windows

Alternatively, if you do not want to force a color, you can use Symbol:

class ui.Symbol(as_unicode, as_ascii)¶
>>> heart = ui.Symbol("❤", "<3")
>>> ui.info("Thanks for using python-cli-ui", heart)
Thanks for using python-cli-ui ❤  # on Linux
Thanks for using python-cli-ui <3  # on Windows

Informative messages¶

ui.info(*tokens, **kwargs)¶

Print an informative message

Parameters:
  • tokens – list of ui constants or strings, like (ui.red, 'this is an error')
  • sep – separator, defaults to ' '
  • end – token to place at the end, defaults to '\n'
  • fileobj – file-like object to print the output, defaults to sys.stdout
  • update_title – whether to update the title of the terminal window
 
>>> ui.info("this is", ui.red, "red")
This is red

Functions below take the same arguments as the info() function

ui.info_section(*tokens, **kwargs)¶

Print an underlined section name

>>> ui.info_section("Section one")
>>> ui.info("Starting stuff")

Section one
------------

Starting stuff
ui.info_1(*tokens, **kwargs)¶

Print an important informative message

>>> ui.info_1("Message")
:: Message
ui.info_2(*tokens, **kwargs)¶

Print an not so important informative message

>>> ui.info_2("Message")
=> Message
ui.info_3(*tokens, **kwargs)¶

Print an even less important informative message

>>> ui.info_3("Message")
* Message
ui.debug(*tokens, **kwargs)¶

Print a debug message.

Messages are shown only when CONFIG["verbose"] is true

>>> ui.debug("Message")
<nothing>

Error messages¶

Functions below use sys.stderr by default:

ui.error(*tokens, **kwargs)¶

Print an error message

>>> ui.error("Message")
Error: message
ui.warning(*tokens, **kwargs)¶

Print a warning message

>>> ui.warning("Message")
Warning: message
ui.fatal(*tokens, **kwargs)¶

Print an error message and call sys.exit

>>> ui.fatal("Message")
Error: message
exit()

Progress messages¶

ui.dot(*, last=False, fileobj=None)¶

Print a dot without a newline unless it is the last one.

Useful when you want to display a progress with very little knowledge.

Parameters:last – whether this is the last dot (will insert a newline) 
>>> for in in rang(0, 5):
>>>     ui.dot()
....<no newline>
>>> ui.dot(last=True)
.....
ui.info_count(i, n, *rest, **kwargs)¶

Display a counter before the rest of the message.

rest and kwargs are passed to info()

Current index should start at 0 and end at n-1, like in enumerate()

Parameters:
  • i – current index
  • n – total number of items
 
>>> ui.info_count(4, 12, message)
* ( 5/12) message
ui.info_progress(prefix, value, max_value)¶

Display info progress in percent.

Parameters:
  • value – the current value
  • max_value – the max value
  • prefix – the prefix message to print
 
>>> ui.info_progress("Done", 5, 20)
Done: 25%

Formatting¶

ui.tabs(num)¶

Compute a blank tab

>>> ui.info("one", "\n",
            ui.tabs(1), "two", "\n",
            ui.tabs(2), "three", "\n",
            sep="")
one
  two
    three
ui.indent(text, num=2)¶

Indent a piece of text.

>>> quote = (
      "First, we take Manhattan\n"
      "Then we take Berlin!"
)
>>> ui.info("John said:", ui.indent(quote))
John said:
   First, we take Manhattan.
   Then we take Berlin!
ui.info_table(data, *, headers=None, fileobj=None)¶
>>> headers=["name", "score"]
>>> data = [
      [(bold, "John"), (green, 10.0)],
      [(bold, "Jane"), (green, 5.0)],
    ]

>>> ui.info_table(data, headers=headers)
name      score
--------  --------
John       10.0
Jane        5.0

Asking for user input¶

ui.read_input()¶

Read input from the user

ui.ask_string(question, default=None)¶

Ask the user to enter a string.

>>> name = ui.ask_string("Enter your name")
:: Enter your name
<john>
>>> name
'john'
ui.ask_choice(input_text, choices, *, func_desc=None)¶

Ask the user to choose from a list of choices.

Returns:the selected choice

func_desc will be called on every list item for displaying and sorting the list. If not given, will default to the identity function.

Will loop until:
  • the user enters a valid index
  • or hits ctrl-c
  • or leaves the prompt empty

In the last two cases, None will be returned

>>> choices = ["apple", "banana", "orange"]
>>> fruit = ui.ask_choice("Select a fruit", choices)
:: Select a fruit
  1 apple
  2 banana
  3 orange
<2>
>>> fruit
'banana'
ui.ask_yes_no(question, default=False)¶

Ask the user to answer by yes or no

>>> with_cream = ui.ask_yes_no("With cream?", default=False)
:: With cream? (y/N)
<y>
>>> with_cream
True

Displaying duration¶

class ui.Timer(description)¶

Display time taken when executing a list of statements.

>>> @ui.Timer("something")
    def do_something():
         foo()
         bar()
# Or:
>>> with ui.Timer("something"):
        foo()
        bar()
* Something took 0h 3m 10s 430ms

Auto-correct¶

ui.did_you_mean(message, user_input, choices)¶

Given a list of choices and an invalid user input, display the closest items in the list that match the input.

>>> allowed_names = ["Alice", "John", "Bob"]
>>> name = ui.ask_string("Enter a name")
>>> if not name in allowed_names:
>>>       ui.did_you_mean("Invalid name", user_input, choices)
:: Enter a name
<Joohn>
Invalid name.
Did you mean: John?

Note: if the list of possible choices is short, consider using ask_choice() instead.

Testing with pytest¶

ui.tests.conftest.message_recorder(request)¶

Start recording messages

Methods

  • stop(): stop recording
  • reset(): clear the list of recorded messages.
  • find(regex) find a message in the list matching the given regular
    expression
from ui.tests.conftest import message_recorder


def foo():
    ui.info("Fooing")


def test_foo(message_recorder):
     foo()
     assert message_recorder.find("Fooing")
